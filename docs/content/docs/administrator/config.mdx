MineAuth manages plugin settings through the `plugins/MineAuth/config.json` file.

## File Structure

Configuration files are located in the `plugins/MineAuth/` directory.

```
plugins/MineAuth/
├── config.json          # Main configuration file
├── clients.json         # OAuth client settings
└── generated/           # Auto-generated files
    ├── privateKey.pem   # RSA private key
    ├── publicKey.pem    # RSA public key
    ├── certificate.pem  # X.509 certificate
    └── jwks.json        # JWK Set
```

## config.json

```json5
{
  "server": {
    "baseUrl": "https://api.example.com",  // Base URL for OIDC issuer and endpoints
    "port": 8080,
    "ssl": {
      "sslPort": 8443,
      "keyStore": "keystore.jks",
      "keyAlias": "MineAuth",
      "keyStorePassword": "password",
      "privateKeyPassword": "password"
    } // Can be null or omitted if SSL is not required
  },
  "jwt": {
    "issuer": "https://api.example.com/",  // JWT token issuer
    "realm": "example.com",
    "privateKeyFile": "privateKey.pem",
    "keyId": "a22c063a-a708-c801-6f92-49f6d53b89b2"  // JWK key ID
  },
  "oauth": {
    "applicationName": "MineAuth",  // Application name displayed on authorization screen
    "logoUrl": "/assets/lock.svg"   // Logo URL displayed on authorization screen
  }
}
```

### Configuration Options

#### server

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `baseUrl` | string | Yes | Base URL for endpoints published in OIDC Discovery. Set to proxy URL if using a reverse proxy |
| `port` | number | Yes | HTTP server port number |
| `ssl` | object | No | SSL configuration (optional) |

#### jwt

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `issuer` | string | Yes | JWT token issuer identifier |
| `realm` | string | Yes | JWT authentication realm name |
| `privateKeyFile` | string | Yes | Private key file path (relative to generated/) |
| `keyId` | string | Yes | JWK key ID (UUID format) |

#### oauth

| Key | Type | Required | Description |
|-----|------|----------|-------------|
| `applicationName` | string | Yes | Application name displayed on OAuth authorization screen |
| `logoUrl` | string | Yes | Logo URL displayed on OAuth authorization screen |

## Auto-generated Files

The `generated/` directory contains cryptographic keys and certificates automatically generated by the plugin.

- **privateKey.pem**: RSA private key used for RS256 signing
- **publicKey.pem**: RSA public key
- **certificate.pem**: Self-signed X.509 certificate
- **jwks.json**: JWK Set (public key in JSON Web Key format)

These files are auto-generated on first startup. If deleted, they will be regenerated, but existing tokens will be invalidated.
