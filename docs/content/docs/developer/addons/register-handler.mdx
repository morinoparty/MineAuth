This document explains how external plugins can register HTTP endpoints using the `MineAuthApi` interface.

## Overview

MineAuth exposes a `MineAuthApi` service that allows external plugins to register their own HTTP endpoints. Endpoints are mounted under `/api/v1/plugins/{namespace}/` and use annotation-based routing similar to Spring MVC.

## Getting Started

### Obtaining MineAuthApi

Obtain the `MineAuthApi` instance through the Bukkit ServicesManager. The `MineAuthApi.get(server)` helper wraps the lookup and returns `null` when MineAuth is not loaded:

```kotlin
class MyPlugin : JavaPlugin() {
    override fun onEnable() {
        val api = MineAuthApi.get(server)
            ?: run {
                logger.severe("MineAuth not found")
                server.pluginManager.disablePlugin(this)
                return
            }

        try {
            // Register all handlers in a single call, under the "shop" namespace
            val registration = api.register(this, "shop", ShopHandler(), UserHandler())
            logger.info("Mounted ${registration.endpoints.size} endpoints under ${registration.basePath}")
        } catch (e: EndpointRegistrationException) {
            logger.severe(e.message)
            server.pluginManager.disablePlugin(this)
        }
    }
}
```

### Creating a Handler Class

Handler classes contain methods annotated with HTTP method annotations. Every endpoint must also declare its access level explicitly with `@Public` or `@Authenticated`:

```kotlin
class ShopHandler {

    @Get("/shops")
    @Public
    suspend fun listShops(): List<ShopData> {
        // Return list of shops
        return shopService.findAll()
    }

    @Get("/shops/{id}")
    @Public
    suspend fun getShop(@Path("id") id: String): ShopData? {
        // Return single shop
        return shopService.findById(id)
    }

    @Post("/shops")
    @Authenticated(permission = "myshop.create")
    suspend fun createShop(@Caller caller: Principal.User, @Body shop: ShopData): ShopData {
        // Create new shop owned by the authenticated player
        return shopService.create(caller.uuid, shop)
    }
}
```

## All-or-Nothing Registration

`register()` validates every endpoint of every handler before mounting anything. If any endpoint fails validation, **no endpoint is mounted** and an `EndpointRegistrationException` is thrown. The exception carries the complete list of typed validation errors in its `errors` property, and its `message` contains a human-readable summary of all of them:

```
MineAuth endpoint registration failed (2 error(s)):
  - ShopHandler#listShops: no access declaration -> annotate with @Public or @Authenticated
  - ShopHandler#createShop: @Body parameter 'shop' is not deserializable: ... -> annotate the type with @Serializable
```

Validation covers, among others:

- Missing or conflicting `@Public`/`@Authenticated` declarations
- Parameters without an annotation, or with conflicting annotations
- `@Path`/`@PlayerParam` names that do not appear as `{segment}` in the route path
- Unsupported parameter types
- `@Body` types that are not `@Serializable`
- `@Caller` nullability or `Principal` subtype mismatches
- Duplicate routes within the namespace
- Invalid or already-claimed namespaces

## Namespaces and Route Generation

The base path is `/api/v1/plugins/{namespace}`. Two overloads are available:

| Overload | Namespace |
|----------|-----------|
| `register(plugin, vararg handlers)` | Derived from `plugin.name.lowercase()` |
| `register(plugin, namespace, vararg handlers)` | Explicit namespace (recommended) |

An explicit namespace must match `[a-z0-9-]{2,32}` and must not be in use by another plugin. Use it to keep URLs short when the plugin name is unwieldy (e.g. plugin `MineAuth-Addon-Vault` registering under `vault`).

### Generated Endpoints

For the handler above registered under namespace `shop`:

| Annotation | Generated Route |
|------------|----------------|
| `@Get("/shops")` | `GET /api/v1/plugins/shop/shops` |
| `@Get("/shops/{id}")` | `GET /api/v1/plugins/shop/shops/{id}` |
| `@Post("/shops")` | `POST /api/v1/plugins/shop/shops` |

### Class-Level Path Prefix

Use `@Route` on the handler class to prefix all endpoint paths in that class:

```kotlin
@Route("/shops")
class ShopHandler {

    @Get("/{id}")           // => GET /api/v1/plugins/shop/shops/{id}
    @Public
    suspend fun getShop(@Path("id") id: String): ShopData? { ... }
}
```

## The Registration Handle

`register()` returns a `MineAuthRegistration` describing what was mounted:

```kotlin
val registration = api.register(this, "shop", ShopHandler())

registration.basePath       // "/api/v1/plugins/shop"
registration.endpoints      // List<RegisteredEndpoint>: method, full path, handler, access info
registration.unregister()   // explicit unregistration (idempotent)
```

Registrations are **automatically unregistered when your plugin is disabled**, so calling `unregister()` manually is rarely necessary. `MineAuthRegistration` also implements `AutoCloseable` (`close()` delegates to `unregister()`).

## Parameter Handling

### Path Parameters

Use `@Path` to extract typed path parameters (`String`, `Int`, `Long`, `Double`, `Float`, `Boolean`, `UUID`):

```kotlin
@Get("/shops/{id}")
@Public
suspend fun getShop(@Path("id") id: Long): ShopData?
```

### Query Parameters

Use `@Query` for typed, individual query parameters. A nullable Kotlin type marks the parameter as optional (resolved to `null` when absent); a non-nullable type makes it required and returns `400 Bad Request` when missing:

```kotlin
@Get("/shops")
@Public
suspend fun searchShops(
    @Query("limit") limit: Int?,      // optional, no manual parsing needed
    @Query("cursor") cursor: String?  // optional
): List<ShopData> {
    val effectiveLimit = limit ?: 10
    // ...
}
```

If the handler genuinely needs to iterate arbitrary query keys, `@QueryMap params: Map<String, String>` is available as an escape hatch.

### Request Body

Use `@Body` to receive a JSON body. The type must be `@Serializable`; the serializer is resolved and validated at registration time:

```kotlin
@Post("/shops")
@Authenticated(permission = "myshop.create")
suspend fun createShop(@Caller caller: Principal.User, @Body shop: ShopData): ShopData {
    return shopService.create(caller.uuid, shop)
}
```

## Complete Example

```kotlin
class ShopHandler : KoinComponent {
    private val shopService: ShopService by inject()

    @Get("/shops")
    @Public
    suspend fun listShops(@Query("limit") limit: Int?): List<ShopData> {
        return shopService.findAll(limit ?: 20)
    }

    @Get("/shops/{id}")
    @Public
    suspend fun getShop(@Path("id") id: String): ShopData? {
        return shopService.findById(id)
    }

    @Post("/shops")
    @Authenticated(permission = "myshop.create")
    suspend fun createShop(
        @Caller caller: Principal.User,
        @Body shop: CreateShopRequest
    ): ShopData {
        return shopService.create(caller.uuid, shop)
    }

    @Put("/shops/{id}")
    @Authenticated(permission = "myshop.edit")
    suspend fun updateShop(
        @Caller caller: Principal.User,
        @Path("id") id: String,
        @Body shop: UpdateShopRequest
    ): ShopData {
        return shopService.update(id, caller.uuid, shop)
    }

    @Delete("/shops/{id}")
    @Authenticated(permission = "myshop.delete")
    suspend fun deleteShop(
        @Caller caller: Principal.User,
        @Path("id") id: String
    ) {
        shopService.delete(id, caller.uuid)
    }
}
```

## Error Handling

Throw `HttpError` — or call the `fail()` helper, whose `Nothing` return type composes well with expressions — to return custom error responses. The optional `code` field provides a machine-readable identifier for client-side branching, and `details` is a `Map<String, String>`:

```kotlin
import party.morino.mineauth.api.http.HttpError
import party.morino.mineauth.api.http.HttpStatus
import party.morino.mineauth.api.http.fail

@Get("/shops/{id}")
@Public
suspend fun getShop(@Path("id") id: String): ShopData {
    return shopService.findById(id)
        ?: fail(
            status = HttpStatus.NOT_FOUND,
            message = "Shop not found",
            code = "shop_not_found",
            details = mapOf("id" to id)
        )
}
```

### Error Response Format

```json
{
    "error": "Shop not found",
    "code": "shop_not_found",
    "details": {
        "id": "123"
    }
}
```

### Returning Either

If your addon already uses [ArrowKt](https://arrow-kt.io/), handler methods may declare `Either<HttpError, T>` as their return type. MineAuth unwraps the value: `Right(value)` is serialized as the response body, `Left(httpError)` is converted to the corresponding error response.

```kotlin
@Get("/shops/{id}")
@Public
suspend fun getShop(@Path("id") id: String): Either<HttpError, ShopData> = either {
    ensureNotNull(shopService.findById(id)) {
        HttpError(HttpStatus.NOT_FOUND, "Shop not found", code = "shop_not_found")
    }
}
```

## Notes on List Responses

Handler methods may return `List<T>` of any `@Serializable` data class, including classes defined in the calling plugin. MineAuth resolves the handler's declared return type when serializing the response, so list element types are serialized correctly without relying on runtime type guessing.
