All bundled addons register their handlers with an explicit URL namespace via `api.register(this, "<namespace>", handler)`, so their endpoints are mounted under short base paths such as `/api/v1/plugins/vault`.

## QuickShop-Hikari Addon

A complete example addon implementation is available in the MineAuth repository:

**[morinoparty/MineAuth/addons/quickshop-hikari](https://github.com/morinoparty/MineAuth/tree/master/addons/quickshop-hikari)**

This addon provides HTTP endpoints for QuickShop-Hikari shop data (namespace: `quickshop`):

| File | Description |
|------|-------------|
| `QuickShopHikariAddon.kt` | Main plugin class |
| `routes/ShopHandler.kt` | HTTP handler with endpoints |
| `data/*.kt` | Data classes for shop information |

### Generated Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/plugins/quickshop/shops` | `@Public` | List all shops (cursor-based pagination) |
| GET | `/api/v1/plugins/quickshop/shops/{shopId}` | `@Public` | Get shop by ID |
| GET | `/api/v1/plugins/quickshop/users/{player}/shops` | `@Authenticated` + `@PlayerParam` | Get own or target player's shops |
| GET | `/api/v1/plugins/quickshop/shops/{shopId}/setting` | `@Authenticated` + `@Caller` | Get shop settings (owner only) |
| POST | `/api/v1/plugins/quickshop/shops/{shopId}/setting` | `@Authenticated` + `@Caller` | Update shop settings (owner only) |

The `users/{player}/shops` endpoint is declared with `@Authenticated(callers = [CallerType.USER, CallerType.SERVICE])` and `@PlayerParam("player")`, which means:
- Users can access their own shops via `/users/me/shops`
- Service accounts can access any player's shops via `/users/Notch/shops` or `/users/<uuid>/shops`

#### Pagination Example

The `/shops` endpoint uses typed optional query parameters — `@Query("cursor") cursor: Long?` and `@Query("limit") limit: Int?` — so no manual parsing is required in the handler:

```kotlin
@Get("/shops")
@Public
suspend fun listAllShops(@Query("cursor") cursor: Long?, @Query("limit") limit: Int?): PaginatedShopsResponse
```

```
GET /api/v1/plugins/quickshop/shops?limit=200
GET /api/v1/plugins/quickshop/shops?cursor=12345&limit=200
```

## Vault Addon

Provides HTTP endpoints for Vault economy operations (namespace: `vault`):

**[morinoparty/MineAuth/addons/vault](https://github.com/morinoparty/MineAuth/tree/master/addons/vault)**

| File | Description |
|------|-------------|
| `VaultAddon.kt` | Main plugin class |
| `routes/VaultHandler.kt` | HTTP handler with endpoints |
| `data/RemittanceData.kt` | Data class for transfer requests |

### Generated Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/plugins/vault/balance/{player}` | `@Authenticated` + `@PlayerParam` | Get own or target player's balance |
| POST | `/api/v1/plugins/vault/send` | `@Authenticated` + `@Caller` | Send money to another player (user tokens only) |

### Handler Code

The actual handler illustrates both authentication styles:

```kotlin
class VaultHandler : KoinComponent {
    private val economy: Economy by inject()

    // ユーザートークンは自分自身のみ、サービストークンは任意のプレイヤーの残高を取得できる
    @Get("/balance/{player}")
    @Authenticated(callers = [CallerType.USER, CallerType.SERVICE])
    suspend fun getMyBalance(@PlayerParam("player") player: OfflinePlayer): BalanceResponse {
        return BalanceResponse(balance = economy.getBalance(player))
    }

    // 自分自身の口座からの送金のみを許可するため、ユーザートークンでの呼び出しのみ許可する
    @Post("/send")
    @Authenticated
    suspend fun sendMoney(
        @Caller caller: Principal.User,
        @Body data: RemittanceData
    ): TransferResponse {
        val player = caller.offlinePlayer
        // 入力検証・残高チェックの後に送金処理を実行し、失敗時はHttpErrorをスローする
        // ...
    }
}
```

The registration in `VaultAddon.kt`:

```kotlin
private fun setupMineAuth(): Boolean {
    return try {
        val registration = mineAuthApi.register(this, "vault", VaultHandler())
        logger.info("Mounted ${registration.endpoints.size} endpoints under ${registration.basePath}")
        true
    } catch (e: EndpointRegistrationException) {
        // 登録は全件失敗（all-or-nothing）のため、検証エラーの詳細をログに出力する
        logger.severe(e.message)
        false
    }
}
```

The balance endpoint uses `@PlayerParam` with the default `SELF_OR_SERVICE` policy:
- Users: `GET /balance/me` to check own balance
- Service accounts: `GET /balance/Notch` or `GET /balance/<uuid>` to check any player's balance

## PureTickets Addon

Provides HTTP endpoints for PureTickets support ticket data (namespace: `tickets`):

**[morinoparty/MineAuth/addons/pure-tickets](https://github.com/morinoparty/MineAuth/tree/master/addons/pure-tickets)**

| File | Description |
|------|-------------|
| `PureTicketsAddon.kt` | Main plugin class (reflection-based API access) |
| `routes/TicketHandler.kt` | HTTP handler with endpoints |
| `data/*.kt` | Data classes for ticket information |

### Generated Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/plugins/tickets/tickets/{player}` | `@Authenticated` + `@PlayerParam` | Get own or target player's tickets |
| GET | `/api/v1/plugins/tickets/tickets/{player}/{id}` | `@Authenticated` + `@PlayerParam` | Get ticket detail with interaction history |

Both endpoints are declared with `@Authenticated(callers = [CallerType.USER, CallerType.SERVICE])` and `@PlayerParam("player")`:
- Users: `GET /tickets/me` to check own tickets
- Service accounts: `GET /tickets/Notch` or `GET /tickets/<uuid>` to check any player's tickets

### Response Examples

`GET /tickets/me`:

```json
{
  "tickets": [
    {
      "id": 1,
      "status": "OPEN",
      "message": "The door at spawn is broken",
      "claimer": null
    }
  ],
  "total": 1
}
```

`GET /tickets/me/1`:

```json
{
  "id": 1,
  "status": "CLAIMED",

```

### Note

PureTickets does not expose a public API for external plugins. This addon uses reflection to access internal services via Guice Injector. Compatible with PureTickets v5.x.

## VotingPlugin Addon

Provides HTTP endpoints for VotingPlugin vote data (namespace: `voting`):

| File | Description |
|------|-------------|
| `VotingPluginAddon.kt` | Main plugin class |
| `routes/VotingPluginHandler.kt` | HTTP handler with endpoints |
| `data/*.kt` | Data classes for vote information |

### Generated Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/plugins/voting/stats/{player}` | `@Authenticated` + `@PlayerParam` | Get own or target player's vote statistics |
