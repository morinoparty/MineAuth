This document provides a comprehensive reference for all annotations available in the MineAuth API. All annotations live in the `party.morino.mineauth.api.annotations` package.

## HTTP Method Annotations

These annotations define the HTTP method and path for your endpoint methods. Exactly one HTTP method annotation must be present per endpoint method.

### @Get

Defines a GET request endpoint.

```kotlin
@Get("/users")
@Public
suspend fun listUsers(): List<UserData>

@Get("/users/{id}")
@Public
suspend fun getUser(@Path("id") id: UUID): UserData?
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `value` | String | The URL path for the endpoint |

### @Post

Defines a POST request endpoint.

```kotlin
@Post("/users")
@Authenticated(permission = "myplugin.users.create")
suspend fun createUser(@Body user: CreateUserRequest): UserData
```

### @Put

Defines a PUT request endpoint.

```kotlin
@Put("/users/{id}")
@Authenticated(permission = "myplugin.users.edit")
suspend fun updateUser(
    @Path("id") id: UUID,
    @Body user: UpdateUserRequest
): UserData
```

### @Delete

Defines a DELETE request endpoint.

```kotlin
@Delete("/users/{id}")
@Authenticated(permission = "myplugin.users.delete")
suspend fun deleteUser(@Path("id") id: UUID)
```

### @Patch

Defines a PATCH request endpoint.

```kotlin
@Patch("/users/{id}")
@Authenticated(permission = "myplugin.users.edit")
suspend fun patchUser(@Path("id") id: UUID, @Body patch: UserPatch): UserData
```

### @Route

Class-level annotation that adds a path prefix to every endpoint in the handler class.

```kotlin
@Route("/shops")
class ShopHandler {

    @Get("/{shopId}")   // => GET /shops/{shopId}
    @Public
    suspend fun getShop(@Path("shopId") shopId: Long): ShopData
}
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `value` | String | The path prefix applied to all endpoints in the class |

## Access Declaration Annotations

Every endpoint method **must** declare exactly one of `@Public` or `@Authenticated`. Registration fails with an `EndpointRegistrationException` if the declaration is missing or if both are present. This prevents endpoints from being exposed unintentionally.

### @Public

Declares an endpoint that requires no authentication.

```kotlin
@Get("/sites")
@Public
suspend fun getVoteSites(): VoteSitesResponse
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `reason` | String | Optional justification for exposing the endpoint publicly; emitted into the OpenAPI documentation |

### @Authenticated

Declares an endpoint that requires a valid access token.

```kotlin
@Get("/balance/{player}")
@Authenticated(callers = [CallerType.USER, CallerType.SERVICE])
suspend fun getBalance(@PlayerParam("player") player: OfflinePlayer): BalanceResponse
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `permission` | String | Required permission node (empty string = authentication only, no permission check). Evaluated only for user tokens; the check fails with `403 player_offline` if the player is offline |
| `callers` | Array&lt;CallerType&gt; | Token types allowed to call the endpoint. Defaults to `[CallerType.USER]`; service tokens must be opted in explicitly with `[CallerType.USER, CallerType.SERVICE]` |

See the [Authentication Guide](/docs/developer/addons/authentication) for details.

## Parameter Annotations

These annotations extract data from the HTTP request. Every handler parameter must carry exactly one parameter annotation.

### @Path

Extracts a single path parameter. The named segment must appear as `{name}` in the route path, otherwise registration fails.

```kotlin
@Get("/shops/{shopId}")
@Public
suspend fun getShop(@Path("shopId") shopId: Long): ShopData?
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `value` | String | The name of the path segment to extract |

**Supported Types:**
- `String`
- `Int`
- `Long`
- `Double`
- `Float`
- `Boolean`
- `UUID`

### @Query

Extracts a single, typed query parameter. The Kotlin type's nullability determines whether the parameter is required:

- **Nullable type** (`Int?`, `String?`, ...) — optional; resolved to `null` when absent
- **Non-nullable type** (`Int`, `String`, ...) — required; `400 Bad Request` when absent

```kotlin
@Get("/shops")
@Public
suspend fun listShops(
    @Query("limit") limit: Int?,       // optional
    @Query("cursor") cursor: String?   // optional
): PaginatedShopsResponse
```

Supported types are the same as `@Path`. Manual parsing such as `params["limit"]?.toInt()` is no longer necessary.

### @QueryMap

Extracts all query parameters as a `Map<String, String>`. This is an escape hatch for endpoints that genuinely iterate arbitrary query keys; prefer typed `@Query` parameters otherwise.

```kotlin
@Get("/search")
@Public
suspend fun search(@QueryMap params: Map<String, String>): SearchResult
```

### @Body

Extracts and deserializes the JSON request body. The type must be `@Serializable`; the serializer is resolved and validated at registration time. See [Serialization & Class Loaders](/docs/developer/addons/serialization) for how this works when your addon bundles its own kotlinx-serialization runtime.

```kotlin
@Post("/shops")
@Authenticated(permission = "myshop.create")
suspend fun createShop(@Body shop: CreateShopRequest): ShopData

// Data class must be @Serializable
@Serializable
data class CreateShopRequest(
    val name: String,
    val price: Double,
    val quantity: Int
)
```

### @Caller

Injects the authenticated request principal (`party.morino.mineauth.api.auth.Principal`).

```kotlin
@Post("/send")
@Authenticated
suspend fun sendMoney(
    @Caller caller: Principal.User,
    @Body data: RemittanceData
): TransferResponse {
    val sender = caller.offlinePlayer
    // ...
}
```

**Bindable types:**

| Type | Accepted tokens | Constraint |
|------|-----------------|------------|
| `Principal` | User and service tokens | — |
| `Principal.User` | User tokens only | Requires `callers = [CallerType.USER]` (the default) |
| `Principal.Service` | Service tokens only | Requires `callers = [CallerType.SERVICE]` |

**Nullability rules (validated at registration):**

- On `@Authenticated` endpoints the parameter must be **non-nullable** (authentication is guaranteed)
- On `@Public` endpoints the parameter must be **nullable** (`Principal?`); it is `null` for anonymous requests

### @PlayerParam

Resolves an `OfflinePlayer` from a path segment. Only usable on `@Authenticated` endpoints, and the segment must appear as `{name}` in the route path. The segment value accepts `me` (the authenticated user), a UUID, or a player name.

```kotlin
@Get("/balance/{player}")
@Authenticated(callers = [CallerType.USER, CallerType.SERVICE])
suspend fun getBalance(@PlayerParam("player") player: OfflinePlayer): BalanceResponse
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `value` | String | Path segment name to resolve from (default: `"player"`) |
| `access` | PlayerAccess | Access policy for the target player (default: `SELF_OR_SERVICE`) |

| `{player}` value | Behavior |
|-------------------|----------|
| `me` | Resolves to the authenticated player (user tokens only) |
| UUID format | Resolves to the player with that UUID |
| Player name | Resolves to the player with that name |

**PlayerAccess policies:**

| Policy | User token | Service token |
|--------|-----------|---------------|
| `SELF_ONLY` | Own data only | Denied |
| `SELF_OR_SERVICE` (default) | Own data only | Any player |
| `ANY_AUTHENTICATED` | Any player | Any player |

See the [Authentication Guide](/docs/developer/addons/authentication) for details.

## Annotation Usage Summary

| Annotation | Target | Description |
|------------|--------|-------------|
| `@Get` / `@Post` / `@Put` / `@Delete` / `@Patch` | Function | Define an endpoint with the corresponding HTTP method |
| `@Route` | Class | Path prefix for all endpoints in the class |
| `@Public` | Function | Declare a public (unauthenticated) endpoint |
| `@Authenticated` | Function | Require authentication; optional `permission` and `callers` |
| `@Path` | Parameter | Extract a typed path parameter |
| `@Query` | Parameter | Extract a typed query parameter (nullable = optional) |
| `@QueryMap` | Parameter | Extract all query parameters as a Map |
| `@Body` | Parameter | Extract and deserialize the JSON body |
| `@Caller` | Parameter | Inject the authenticated `Principal` |
| `@PlayerParam` | Parameter | Resolve an `OfflinePlayer` from a path segment (me/UUID/name) |

## Choosing the Right Access Declaration

```
Does the endpoint need authentication?
├── No → @Public
│   └── Personalize for logged-in users? → add @Caller caller: Principal?
└── Yes → @Authenticated
    ├── Operates on a player given in the path?
    │   └── Use @PlayerParam with /{player} in the path
    │       ├── Service accounts should reach any player? → callers = [USER, SERVICE] (default access = SELF_OR_SERVICE)
    │       └── Strictly self-service? → access = PlayerAccess.SELF_ONLY
    ├── Acts on behalf of the caller (write/action)?
    │   └── Use @Caller caller: Principal.User (default callers = [USER])
    └── Needs a Bukkit permission? → @Authenticated(permission = "...")
```
