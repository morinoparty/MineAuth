MineAuth provides interactive API documentation through OpenAPI 3.1.0 specification and Scalar UI. The OpenAPI specification is dynamically generated by the plugin, including all registered addon endpoints.

## Endpoints

The REST API is documented using OpenAPI 3.1.0 specification, rendered with [Scalar](https://github.com/scalar/scalar).

Access the documentation through the following endpoints on your MineAuth server:

| Endpoint | Description |
|----------|-------------|
| `/scalar` | Interactive API documentation UI (Scalar) |
| `/openapi` | OpenAPI 3.1.0 specification in JSON format |

## Scalar UI

Access the interactive API documentation at:

```
http://localhost:8080/scalar
```

Scalar provides:

- Interactive API explorer
- Request/response examples
- Authentication testing with OAuth2
- Schema visualization

### Available Endpoint Categories

The OpenAPI specification documents the following endpoint categories:

| Category | Description |
|----------|-------------|
| **OAuth2** | Authorization, token, userinfo, and revocation endpoints |
| **Discovery** | OpenID Connect discovery and JWKs endpoints |
| **Commons** | Server information endpoints (players, plugins) |
| **Plugins** | Dynamically registered addon endpoints |

## OpenAPI Specification

You can retrieve the raw OpenAPI specification from your running server:

```bash
curl http://localhost:8080/openapi
```

The specification includes:

- All dynamically registered addon endpoints
- OAuth2 security scheme definitions
- Request/response schemas
- Parameter definitions

## Dynamic Endpoint Discovery

MineAuth automatically generates OpenAPI documentation for all registered addons. When an addon registers endpoints via `MineAuthApi.register()` using `@Get`, `@Post`, etc., they are automatically included in the OpenAPI specification.

### Example

If the Vault addon registers these endpoints under the `vault` namespace:

```kotlin
@Get("/balance/{player}")
@Authenticated(callers = [CallerType.USER, CallerType.SERVICE])
suspend fun getMyBalance(@PlayerParam("player") player: OfflinePlayer): BalanceResponse

@Post("/send")
@Authenticated
suspend fun sendMoney(
    @Caller caller: Principal.User,
    @Body data: RemittanceData
): TransferResponse
```

They will appear in the OpenAPI spec as:

```json
{
  "paths": {
    "/api/v1/plugins/vault/balance/{player}": {
      "get": {
        "summary": "Get My Balance",
        "parameters": [
          {
            "name": "player",
            "in": "path",
            "required": true,
            "schema": { "type": "string" },
            "description": "Player identifier: 'me', UUID, or player name"
          }
        ],
        "security": [{"oauth2": []}],
        "responses": {
          "200": {
            "content": {
              "application/json": {
                "schema": {
                  "type": "object",
                  "properties": {
                    "balance": {"type": "number", "format": "double"}
                  }
                }
              }
            }
          }
        }
      }
    }
  }
}
```

## Schema Generation

MineAuth automatically generates JSON schemas from Kotlin data classes:

| Kotlin Type | OpenAPI Type |
|-------------|--------------|
| `String` | `string` |
| `Int` | `integer` (format: int32) |
| `Long` | `integer` (format: int64) |
| `Double` | `number` (format: double) |
| `Boolean` | `boolean` |
| `UUID` | `string` (format: uuid) |
| `List<T>` | `array` |
| `Map<K, V>` | `object` |
| `@Serializable` class | `object` with properties |
| `enum class` | `string` with enum values |

Typed `@Query` parameters are documented with their declared type; nullable Kotlin types appear as optional parameters, non-nullable types as required.

## Security

Endpoints declared with `@Authenticated` are marked with OAuth2 security requirements in the OpenAPI spec. Endpoints declared with `@Public` carry no security requirements; if a `reason` was provided on the `@Public` annotation, it is included in the endpoint documentation.

## Public API Surface

The `party.morino.mineauth:api` artifact exposes the following types for addon developers:

| Type | Package | Description |
|------|---------|-------------|
| `MineAuthApi` | `...api` | Entry point; obtained via `MineAuthApi.get(server)`, registers handlers |
| `MineAuthRegistration` | `...api` | Registration handle: `basePath`, `endpoints`, `unregister()` |
| `RegisteredEndpoint` | `...api` | Mounted endpoint metadata (method, full path, access info) |
| `EndpointRegistrationException` | `...api` | Thrown by `register()` with the full list of validation errors |
| `RegistrationError` | `...api` | Sealed hierarchy of typed validation errors |
| `CallerType` | `...api` | `USER` / `SERVICE` token types for `@Authenticated(callers = ...)` |
| `PlayerAccess` | `...api` | `SELF_ONLY` / `SELF_OR_SERVICE` / `ANY_AUTHENTICATED` policies for `@PlayerParam` |
| `HttpMethod` | `...api` | HTTP method enumeration |
| `Principal`, `Principal.User`, `Principal.Service` | `...api.auth` | Authenticated caller injected via `@Caller` |
| `HttpError`, `fail()` | `...api.http` | Error responses with optional machine-readable `code` and `details` |
| `HttpStatus` | `...api.http` | HTTP status enumeration (200–503) |
| `@Get` / `@Post` / `@Put` / `@Delete` / `@Patch` | `...api.annotations` | HTTP method annotations |
| `@Route` | `...api.annotations` | Class-level path prefix |
| `@Public` / `@Authenticated` | `...api.annotations` | Mandatory access declarations |
| `@Path` / `@Query` / `@QueryMap` / `@Body` / `@Caller` / `@PlayerParam` | `...api.annotations` | Parameter annotations |

- **Code Generators** - Generate client SDKs
- **API Testing Tools** - Automated testing
